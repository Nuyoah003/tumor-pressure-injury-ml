## 一、项目背景与目标

### 1.1 背景

压力性损伤（Pressure Injury, PI）是肿瘤患者住院期间常见的并发症之一，严重影响患者生活质量和治疗效果。基于 Braden 量表的风险评估是临床预防的重要手段，但传统人工评估存在主观性强、效率低等问题。本项目旨在开发基于患者临床数据的机器学习模型，实现压力性损伤风险的自动化预测。

### 1.2 目标

基于肿瘤患者的 24 项临床特征变量，构建预测模型对患者的压力性损伤风险进行 **4 级有序分类预测**：

| 风险等级 | Braden 评分 | 临床含义 |
|----------|-------------|----------|
| 0 - 无风险 | ≥ 19 分 | 无需特殊干预 |
| 1 - 低风险 | 15-18 分 | 常规预防措施 |
| 2 - 中风险 | 13-14 分 | 加强预防措施 |
| 3 - 高风险 | ≤ 12 分 | 重点预防干预 |

## 三、建模方法

### 3.1 数据预处理流程

```
原始数据 → 变量编码 → 分层划分(70:30) → 标准化 → 模型训练
                         ↑
                    SMOTE 重采样
                    (仅应用于训练集)
```

**预处理步骤：**

1. **变量编码**
   - 连续变量：保留原始数值
   - 二分类变量：0/1 编码
   - 名义变量（肿瘤类型）：One-hot 编码，删除"其他"类作为参照组，生成 14 维哑变量

2. **数据划分**
   - 采用分层随机划分（Stratified Shuffle Split）
   - 训练集：70%（70,352 条），测试集：30%（30,152 条）
   - 随机种子：42（保证实验可复现）

3. **特征标准化**
   - 对连续变量（年龄、住院时长、白蛋白、BMI）进行 Z-score 标准化
   - 在训练集上拟合，在测试集上变换（避免数据泄露）

### 3.2 SMOTE 重采样技术

针对数据类别极度不平衡问题，采用 **SMOTE（Synthetic Minority Over-sampling Technique）** 合成少数类过采样技术：

- **原理：** 在少数类样本的 K 个最近邻之间进行线性插值，生成新的合成样本
- **策略：** 采用 moderate 中等策略，将每个少数类增采样至多数类的 40%
- **关键参数：** K=5（5 近邻），random_state=42
- **仅应用于训练集**，测试集保持原始不平衡分布

**SMOTE 重采样效果：**

| 风险等级 | 训练集（原始） | 训练集（SMOTE后） | 增量 |
|----------|----------------|-------------------|------|
| 无风险（0） | 67,393 | 67,393 | 不变 |
| 低风险（1） | 2,172 | 26,957 | +24,785 |
| 中风险（2） | 374 | 26,957 | +26,583 |
| 高风险（3） | 413 | 26,957 | +26,544 |
| **合计** | **70,352** | **148,264** | +77,912 |

### 3.3 模型构建

本项目构建了 4 个对比模型，覆盖传统统计方法与主流机器学习方法：

#### 模型 1：有序逻辑回归（基线）

- **算法：** 累积 Logit 有序逻辑回归（Proportional Odds Model）
- **工具包：** statsmodels OrderedModel
- **特点：** 充分利用目标变量的有序性信息，直接建模累积概率
- **训练数据：** 原始不平衡训练集（无 SMOTE）

#### 模型 2：有序逻辑回归 + SMOTE

- **算法：** 同模型 1，但在 SMOTE 重采样后的训练集上拟合
- **目的：** 评估 SMOTE 对有序逻辑回归的提升效果

#### 模型 3：XGBoost + SMOTE + 类别权重

- **算法：** XGBoost 梯度提升树（多分类）
- **工具包：** xgboost XGBClassifier
- **类别权重：** 基于原始训练集类别分布的逆频率加权
  - 无风险：1.04 | 低风险：32.31 | 中风险：187.71 | 高风险：169.82
- **超参数：**
  - n_estimators=300, max_depth=6, learning_rate=0.1
  - subsample=0.8, colsample_bytree=0.8
  - objective="multi:softprob", eval_metric="mlogloss"

#### 模型 4：LightGBM + SMOTE + 类别权重

- **算法：** LightGBM 梯度提升树（多分类）
- **工具包：** lightgbm LGBMClassifier
- **类别权重：** is_unbalance=True（自动调整）
- **超参数：**
  - n_estimators=300, max_depth=6, learning_rate=0.1
  - subsample=0.8, colsample_bytree=0.8
  - objective="multiclass", num_class=4

## 四、评估指标体系

根据甲方要求，采用以下多维度评估指标体系：

### 4.1 核心指标

| 指标 | 说明 | 适用场景 |
|------|------|----------|
| **混淆矩阵** | 展示各类别的预测正确数与错误数 | 直观了解分类细节 |
| **多分类 AUC** | One-vs-Rest 策略计算各类别 AUC，取 macro/weighted 平均 | 综合衡量区分能力 |
| **Cohen's Kappa** | 衡量预测结果与真实结果的一致性，校正随机一致性 | 评估分类器整体一致性 |

### 4.2 辅助指标

| 指标 | 说明 |
|------|------|
| **Macro F1** | 各类别 F1 的算术平均，平等对待每个类别 |
| **Weighted F1** | 各类别 F1 的加权平均（按样本量加权） |
| **G-mean** | 各类别召回率的几何平均，衡量各类别均衡识别能力 |
| **各类别 Precision / Recall / F1** | 逐类别的精确率、召回率、F1 值 |

### 4.3 评估原则

- **所有模型均在原始不平衡测试集上评估**（30,152 条），确保评估结果的客观性
- 使用统一的随机种子和数据划分，保证模型间公平对比

## 五、实验设计与模型对比

### 5.1 实验目的

针对数据类别极端不平衡这一核心挑战，评估以下优化策略的效果：

1. 基线模型（无任何优化）
2. SMOTE 重采样对有序逻辑回归的提升
3. SMOTE + 类别权重 + 树模型（XGBoost/LightGBM）的综合优化

### 5.2 实验流程

```
清洗后数据.xlsx
    │
    ├── 编码 + 分层划分(70:30) + 标准化
    │       │
    │       ├── 训练集(原始) ──→ 模型1: 有序逻辑回归(基线)
    │       │
    │       ├── 训练集(SMOTE) ──→ 模型2: 有序逻辑回归+SMOTE
    │       │                   ├──→ 模型3: XGBoost+SMOTE+类别权重
    │       │                   └──→ 模型4: LightGBM+SMOTE+类别权重
    │       │
    │       └── 测试集(原始) ──→ 统一评估全部模型
    │
    └── 输出: 混淆矩阵 + ROC曲线 + Kappa + AUC + F1 + 对比图表
```

## 七、结果分析与讨论

### 7.1 基线模型的问题

基线有序逻辑回归模型的核心问题：

1. **极端偏向多数类：** 99.8% 的无风险样本被正确识别，但中风险和高风险的召回率均为 0%。
2. **Kappa 仅为 0.078：** 属于"极弱"一致性水平（< 0.20），模型几乎没有实际的分类能力。
3. **根本原因：** 95.8% 的无风险类别主导了模型训练，模型学习到"全部预测为无风险"即可获得 95.8% 的准确率，但完全丧失了对风险类别的识别能力。

### 7.2 SMOTE 优化的效果

SMOTE 重采样显著改善了模型对风险类别的识别：

- **Kappa 提升：** 从 0.078 提升至 0.210（提升 169%），从"极弱"进入"弱"一致性水平。
- **高风险召回率：** 从 0% 提升至 57.1%（有序逻辑回归+SMOTE）。
- **低风险召回率：** 从 4.0% 提升至 58.8%（XGBoost+SMOTE）。
- **G-mean 提升：** 从 0.199 提升至 0.402（XGBoost+SMOTE），表明各类别识别更为均衡。

### 7.3 各模型优劣势分析

| 模型 | 优势 | 劣势 |
|------|------|------|
| 有序逻辑回归+SMOTE | Kappa 最高（0.210），高风险召回率最高（57.1%） | 低风险精确率偏低（11.4%），无风险误判较多 |
| XGBoost+SMOTE | G-mean 最高（0.402），低风险召回率最高（58.8%） | 高风险召回率偏低（26.6%），整体 Kappa 一般 |
| LightGBM+SMOTE | AUC 最高（0.912），Weighted F1 最高（0.944） | 风险类召回率最低，对少数类改善有限 |

### 7.4 数据层面的限制

当前 Kappa 仍处于"弱"水平（< 0.21），根本原因在于数据层面的结构性限制：

1. **类别极端不平衡：** 中风险仅 534 条（0.5%）、高风险仅 590 条（0.6%），即使使用 SMOTE 合成样本，合成数据的特征分布可能与真实分布存在差异。
2. **特征区分力有限：** 当前的 24 个临床特征对于区分"低风险 vs 中风险 vs 高风险"的细粒度差异可能信息量不足。
3. **比例优势假设不成立：** 有序逻辑回归的前提假设（比例优势假设）经检验不成立（平均变异系数 = 1.06），表明各类别之间的特征效应存在差异，单一有序模型难以充分捕捉。

## 八、结论与后续建议

### 8.1 主要结论

1. **数据质量确认：** 清洗后数据集质量良好（100,504 条，无重复、无缺失），可支持建模分析。
2. **基线模型性能不足：** 原始有序逻辑回归 Kappa=0.078，无法有效识别风险患者。
3. **SMOTE 优化有效但有限：** 经 SMOTE 重采样和树模型优化后，Kappa 提升至 0.210，风险类召回率显著改善，但仍未达到理想的分类性能水平。
4. **各类别 AUC 均较高（> 0.86）：** 模型具备一定的概率排序能力，但转化为硬分类时受阈值和类别不平衡影响较大。

### 8.2 后续优化建议

如果甲方对当前模型性能（Kappa ≈ 0.21）有更高要求，建议考虑以下方向：

| 方向 | 具体措施 | 预期效果 |
|------|----------|----------|
| **A. 简化分类任务** | 将 4 类合并为 2 类（无风险 vs 有风险），可显著提升 Kappa | Kappa 预计可达 0.4-0.6 |
| **B. 增加风险样本** | 收集更多中风险和高风险患者数据 | 从根本上缓解类别不平衡 |
| **C. 阈值优化** | 根据临床需求调整分类阈值（如提高灵敏度） | 可定制风险预警灵敏度 |
| **D. 深度学习方法** | 尝试神经网络等更复杂模型 | 可能捕捉非线性交互效应 |
| **E. 特征工程** | 增加交互特征、多项式特征 | 提升特征区分力 |
