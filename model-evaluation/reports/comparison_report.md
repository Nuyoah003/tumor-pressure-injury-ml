## 临床适用场景分析

| 场景 | 推荐模型 | 原因 |
|------|:------:|------|
| 常规临床辅助决策 | **Model A** | 整体一致性好（Kappa=0.286），误报率低，不会过度占用医疗资源 |
| 高风险患者筛查 | **Model B** | 高风险召回率 61%，是 Model A 的 4 倍，适合"宁可误报不漏检" |
| 论文投稿 / 学术报告 | **Model A** | 综合指标全面领先，方法创新性更高（序数代价 + 软投票集成） |
| 护理资源分配 | **Model A** | 精准率更高，预测为风险的患者更可能真的需要干预 |
| 入院初筛 | **Model B** | 不漏检高风险患者，后续可结合人工复核 |

### 组合使用建议

在实际临床部署中，可采用**两阶段策略**：
1. **第一阶段**（Model B 筛查）：所有入院患者先用 MLP 模型快速筛查，标记出所有可能的风险患者（高召回）
2. **第二阶段**（Model A 精判）：Model B 标记为有风险的患者，再用 Model A 做精确判断，排除误报（高精确率）

## 结论

1. **综合性能最优**：Model A（有序逻辑回归集成模型）在 Accuracy、Kappa、Macro F1、Weighted F1 等核心综合指标上全面领先 Model B，是论文投稿和常规临床应用的首选方案。

2. **Model B 的独特价值**：Model B 的 G-mean（0.53 vs 0.33）和高风险召回率（61.0% vs 15.3%）显著优于 Model A，表明神经网络在少数类识别上具有天然的平衡能力，适合"不放过任何一个高风险患者"的筛查场景。

3. **方法互补**：两模型的表现差异反映了统计模型（保守、整体最优）与神经网络（激进、少数类平衡）的本质差异，两者的组合使用可以实现"筛查不漏检 + 精判少误报"的临床最优策略。

4. **后续优化方向**：
   - Model A 需提升高风险召回率（当前仅 15.3%），可尝试调整代价矩阵中高风险类的惩罚权重
   - Model B 需降低无风险类别的假阳性率，可尝试决策阈值调优或样本后处理
   - 两模型融合（stacking / 级联）值得探索
